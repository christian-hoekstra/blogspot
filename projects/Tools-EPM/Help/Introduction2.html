manageOPatch.cmd|sh

Manages OPatch inventory contents from a single Oracle Home, or all Oracle Homes on the system. You can create HTML or XML based reports, and compare two inventories by printing the differances of patches installed.


Features

  • No credentials are required in order to use this utility.
  • Create HTML or XML based inventory report which includes:
    • Basic System information;
    • Products installed;
    • Components installled per product;
    • Patches installed including their details.
  • You can manually specify an specific Oracle home
  • Automaticly detect all Oracle homes on the system create a report for each
  • Reports are automaticly stored in the "\Oracle\Middleware\user_projects\<epmsystem>\diagnostics\reports" folder.
  • In case of a single report, the report is automaticly openen in the default browser after report creation.
  • Compare to XML based reports, and print the difference in patch level.


Limitations

  • Read access is required to the central OPatch inventory in automatic mode.
  • Dependend on a valid and correct OPatch local inventory.
  • Limited to patch number only. Patch descriptions are not stored in OPatch inventories.
  • Location where the reports are generated, cannot be changed.
  • Export formats are limited to HTML and XML.
