lcmReportManager.cmd|sh

Allows you to delete all or selectivly delete LCM migration reports through the command-line. Since EPM 11.1.2.3, this option has been removed from the web interface. A migration report, per default, is purged after 30 days. However in some cases, it makes sense to manuually delete migration reports.


Features

  • No credentials are required in order to use this utility.
  • The option to delete all reports, or all reports created prior a specified amount of days.


Limitations

  • Must be executed from a system that has Foundation Shared Services installed.
  • Not possible to selectivly delete individual reports.
  • No backup options.
