cryptTool.cmd|sh

Allows you to encode and decode password strings compatible with Shared Services through the command-line. In addition it allows you to retrieve the security key from Shared Services (decrypted) or use a security key retrieved from another EPM implementation.


Features

  • No credentials are required in order to use this utility.
  • Create encrypted password strings.
  • Descrypt encrypted password strings e.g. stored in Shared Services registry.
  • Encrypt and descrypt password strings using a security key retrieved from another EPM implementation.
  • Retrieve the security used by Shared Services from a specific EPM implementation.
  • When new security keys are generated from Shared Services, the new keys are automaticly used on next execution.
  • Supports the enclosure of encryted password strings in case the string contains special characters.


Limitations

  • Must be executed from a system that has Foundation Shared Services installed.
  • Only password created by Shared Services (registry) can be decrypted.
  • Created encrypted passwords are only compatible with Shared Services (registry) on the EPM implementation where it was encrypted.
  • Password from native users cannot be decrypted, they are hashed which is a oneway process.
